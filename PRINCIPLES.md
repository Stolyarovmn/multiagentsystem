# PRINCIPLES

## P-01 Data Honesty

Do not present unverified assumptions as facts.

## P-02 Credentials From Environment Only

Tokens, keys and passwords must not appear in tasks, dispatch envelopes, logs or committed docs.

## P-03 Irreversible Actions Need Review

Deletes, deploys, sends, price changes and other irreversible actions require a safety gate.

## P-04 Build Only For Real Pain

Do not add components because they are interesting. Add them because a real workflow needs them.

## P-05 Simple And Inspectable First

Files, bash, tmux and explicit contracts are preferred until scale proves otherwise.

## P-06 Dispute As Artifact

A conceptual disagreement is formal only when recorded as a dispute artifact.

## P-07 Speak Only For Your Role

Do not invent another role's position or approval.

## P-08 Fresh-Agent Continuity

A new agent must be able to continue from committed docs plus local runtime artifacts. If hidden chat is required, the process is broken.

## P-09 Target-Pack Over Chat History

For target projects, store paths, read order, anchors and rules, not old conversations.

## P-10 System Decisions Leave A Trace

Changes to roles, lifecycle, runtime or policy must leave a task, issue, PR, dispute or roadmap entry.
